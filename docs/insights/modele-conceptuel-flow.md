# Modèle Conceptuel FLOW

## Modèle cible

Party
→ Agreement
→ Demand
→ Decision
→ Execution

## Party
Acteur impliqué dans une relation avec l'entreprise.

## Agreement
Cadre d'engagement applicable.

## Demand
Demande adressée à l'entreprise.

## Decision
Arbitrage permettant de satisfaire ou non la demande.

## Execution
Réalisation effective de la décision.

## Importance

Ce modèle permet de dépasser les découpages historiques Achat / B2B / B2C.
